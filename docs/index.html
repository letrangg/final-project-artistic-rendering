Milestone Status Report

1) What we have done so far: We managed to research into computer simulated watercolor and found a reliable method (Lattice Boltzmann Method) to adopt. (O'Brien, Patrick Michael (2008). A framework for digital watercolor. Master's thesis, Texas A&M University. Texas A&M University.) We structured our project into two main parts: GUI and OpenGL shaders. We will implement most of the computation needed into separate OpenGL shaders and call those shaders in a main program ran by Processing to produce our GUI interface. Under this structure, we managed to build a very simple drawing software along with a simple shader to model the brush strokes. We also have some of the pieces in place to create the full framework for managing, updating, and rendering the texture data in Processing

2) The preliminary results: So far, we’ve managed to build a simple drawing software using Processing’s built-in shaders. A shader program to model the brush strokes were also written in GLSL. The results can be found here: video link, slides link

3) Accordance to Plan: In our original proposal, we structured our project into implementing different pigments (watercolor, paint, etc..). However, we received feedback from the TA’s noting that multiple pigments would probably be out of the time scope for this project. We then changed our project structure into implementing only watercolor instead of multiple pigments. Additionally, we will not be simulating multiple materials to be drawn on and will just use a paper-like canvas.

4) Updated Plan: To efficiently implement computer simulated watercolor, we made plans to separate our computation load into different shaders. Once we have all the equations built into the shader programs, we plan to write a main program in Processing to call the shaders to produce a canvas-like GUI. Doing so, our plans naturally fell into implementing 1~2 shaders per week, along with building the main program as we go. So far, we are slightly behind schedule in that we only managed to implement one shader program (the brush shader), and we have 3 more shaders to go (general update, pigment layer update, net color calculation).
